# Locus v0.2.0 — Configuration & Completeness

> Date: 2026-02-23
> Author: Magnifico4625 + Claude
> Status: Approved

## Theme

Finish what v0.1.0 started. Fix the broken `LOCUS_CAPTURE_LEVEL` server gap, activate the dormant compressor, add config visibility, improve FTS5 UX, and clean up sentinel values. No new parsers, no external APIs.

## Scope — 5 Changes

### 1. Wire `LOCUS_CAPTURE_LEVEL` in server.ts

**Problem:** Env var is read only by `hooks/post-tool-use.js`. The MCP server ignores it — `server.ts` line 67 uses `{ ...LOCUS_DEFAULTS }` with a comment "env overrides can be layered here later".

**Solution:** Read `process.env.LOCUS_CAPTURE_LEVEL` in `server.ts` and override `config.captureLevel` if valid. Mirror the existing `LOCUS_LOG` pattern (line 238).

**Validation:** `metadata | redacted | full` — reject invalid values, log a warning, keep default.

**Files:** `src/server.ts` (modify), test file (create/modify)

### 2. `memory_compact` tool + `/locus:compact` skill

**Problem:** `MemoryCompressor` class exists (`src/memory/compressor.ts`) but is never invoked. Episodic memory grows unbounded.

**Solution:** New MCP tool `memory_compact` with simple time-based truncation:
- Delete episodic entries older than `maxAgeDays` (default: 30)
- Always keep the last `keepSessions` sessions (default: 5) regardless of age
- Return `{ deletedEntries: number, remainingSessions: number, remainingEntries: number }`

**No LLM summarization.** Pure SQL DELETE with date/session guards.

**Parameters:**
- `maxAgeDays?: number` — default 30
- `keepSessions?: number` — default 5

**Skill:** `skills/compact/SKILL.md` → `/locus:compact`

**Files:**
- `src/tools/compact.ts` (create)
- `src/server.ts` (modify — register tool)
- `tests/tools/compact.test.ts` (create)
- `skills/compact/SKILL.md` (create)

### 3. `memory_config` tool + `/locus:memory-config` skill

**Problem:** Users can't see what config values are active or where they came from.

**Solution:** New MCP tool `memory_config` — read-only, no parameters. Returns a table:

```
Setting              Value       Source
captureLevel         metadata    default
maxScanFiles         10000       default
maxFileSize          1048576     default
compressionThreshold 10000       default
logLevel             error       env (LOCUS_LOG)
fts5Available        true        detected
```

Each row: setting name, current value, source (`default` | `env` | `detected`).

**Skill:** `skills/memory-config/SKILL.md` → `/locus:memory-config`

**Files:**
- `src/tools/config.ts` (create)
- `src/server.ts` (modify — register tool)
- `tests/tools/config.test.ts` (create)
- `skills/memory-config/SKILL.md` (create)

### 4. FTS5 fallback documentation

**Problem:** When FTS5 is unavailable, search silently degrades to LIKE. Users don't know why search results are worse.

**Solution:**
- `memory_doctor`: Change FTS5 check message from generic "FTS5 not available" to explicit "Search uses LIKE fallback (slower, less accurate). This depends on your Node.js SQLite build."
- `memory_status`: Add `searchEngine: "FTS5" | "LIKE fallback"` to output
- `README.md`: Add a note in the Configuration section about FTS5 detection

**Files:**
- `src/tools/doctor.ts` (modify)
- `src/tools/status.ts` (modify)
- `README.md` (modify)
- Tests for modified tools (modify)

### 5. Cleanup: NULL sentinels + denylisted test

**Problem:**
- `storeSkippedEntry` uses `'typescript'`, `0`, `'high'` as sentinel values for skipped files. These are misleading — a skipped file is not TypeScript with high confidence.
- The `'denylisted'` skip reason path has no test coverage.

**Solution:**
- Change `storeSkippedEntry` to use `NULL` for `language`, `confidence`, and `confidenceReason` on skipped entries
- Add test: scan a fixture with a `.key` file → verify it gets `skippedReason: 'denylisted'`

**Files:**
- `src/scanner/index.ts` (modify)
- `tests/scanner/scan-project.test.ts` (modify)
