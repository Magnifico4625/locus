# Locus v3.5.1 Release Notes

**Release focus:** Codex npm install hotfix.

`v3.5.1` fixes a Windows/Codex startup issue found immediately after the `v3.5.0` npm release.

## What Was Fixed

When Codex was launched from inside the Locus repository, the package-owned MCP command:

```toml
command = "npx.cmd"
args = ["-y", "locus-memory@3.5.0", "mcp"]
```

could fail with:

```text
'locus-memory' is not recognized as an internal or external command
```

Root cause: npm executed from a project whose root package is also named `locus-memory`, so `npx` resolved against the local workspace context instead of the registry package.

## Fix

The installer now writes a safe working directory into the Codex MCP config:

```toml
[mcp_servers.locus]
command = "npx.cmd"
args = ["-y", "locus-memory@3.5.1", "mcp"]
cwd = "C:\\Users\\<you>\\.codex"
```

This keeps recurring MCP startup away from project-local npm workspace resolution.

## Validation

- reproduced the failure from the Locus repository root
- verified the same npm runtime stays alive when launched with `cwd=$CODEX_HOME`
- added tests for config `cwd` rendering and install-time config patching
- verified local Codex config shows `cwd: C:\Users\Admin\.codex`

## Upgrade

Run:

```bash
npx -y locus-memory@latest install codex
```

Then restart Codex.
