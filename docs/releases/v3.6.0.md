# Locus v3.6.0 Release Notes

**Release focus:** Track C richer Codex conversational recall.

`v3.6.0` turns the Codex path from "import and diagnostics work" into fixture-backed useful recall. The validated path remains Codex CLI with `redacted` capture. Desktop / extension parity is still reported honestly as unverified until tested in that surface.

## Install

```bash
npx -y locus-memory@latest install codex
```

Then restart Codex and verify:

```bash
npx -y locus-memory@latest doctor codex
```

## What Changed

- `memory_recall` is now the summary-first workflow for questions about previous work, prior decisions, dated context, and user workflow style.
- Track C acceptance fixtures prove redacted recall for:
  - Russian "what did we do yesterday?" style questions
  - capture-strategy decisions
  - rejected hook-first rationale
  - user workflow/style preferences
  - npm install error context
  - next steps
  - validation facts
- Recall ambiguity is inspectable through `candidateGroups`; agents should ask a focused clarification question instead of guessing.
- Durable extraction now covers accepted decisions, rejected alternatives, next steps, validation facts, workflow style, and related topic keys.
- `memory_review` and audit docs make stored durable memories easier to inspect before cleanup.
- Codex skill instructions now distinguish `metadata`, `redacted`, and `full` clearly.
- The documented one-command installer path works without `--yes`, and Codex MCP ownership checks now run against the resolved `CODEX_HOME`.
- Explicit memories saved through `memory_remember` are now available to `memory_recall`.

## Capture Modes

| Mode | Product meaning |
|------|-----------------|
| `metadata` | Diagnostics-first mode. It proves import health, but conversational recall is limited. |
| `redacted` | Recommended rich recall mode for Codex. Stores bounded, filtered, best-effort-redacted snippets and durable high-value facts. |
| `full` | Maximum recall. Requires an explicit privacy warning and should never be described as risk-free. |

## Known Boundaries

- Codex CLI is the primary validated path.
- Codex desktop / extension parity remains unverified until direct testing proves the same behavior in that surface.
- `metadata` is not strong conversational memory.
- `redacted` redaction is best-effort; arbitrary free-form text can never be guaranteed perfectly secret-free.
- Codex hooks are optional future freshness triggers. Canonical recall remains JSONL transcript import plus the shared inbox/core ingest path.

## Upgrade Notes

Existing Codex users should rerun the installer after publication to refresh the packaged skill and MCP config:

```bash
npx -y locus-memory@latest install codex
```

Users who want practical conversational recall should keep both capture gates aligned:

```toml
[mcp_servers.locus.env]
LOCUS_CODEX_CAPTURE = "redacted"
LOCUS_CAPTURE_LEVEL = "redacted"
```
