# Locus v3.4.0

## Summary

`v3.4.0` is the Codex memory trust release. It closes the gap found during real local testing: Codex events were being discovered, but useful live recall was not reliable enough. The release validates practical Codex CLI recall with `redacted` capture and keeps documentation honest about what is still unverified.

## Highlights

- Codex CLI `0.123.0` JSONL rollout files are now parsed correctly when events are wrapped in `payload`.
- `memory_search` can find live Codex dialogue decisions in the `conversation` layer without explicit `memory_remember`.
- `memory_recall` now searches matching conversation events beyond the default recent timeline window.
- Track A acceptance docs now describe `redacted` as the recommended practical recall mode.
- `memory_status` and `memory_doctor` continue to report Codex desktop / extension parity as unverified until that surface is tested.

## Upgrade Notes

- Recommended Codex recall configuration:

```toml
[mcp_servers.locus.env]
LOCUS_CODEX_CAPTURE = "redacted"
LOCUS_CAPTURE_LEVEL = "redacted"
```

- `metadata` remains safe for diagnostics-first setups, but it is intentionally limited for conversational recall.
- `full` remains explicit warning territory because it stores the most conversation content locally.
- If several Codex events match the same question, `memory_recall` may return `needs_clarification`; this is expected and inspectable.

## Local Validation Summary

Validated locally before GitHub push:

- `npm test -- packages/codex/tests`
- `npm test -- packages/core/tests/tools/recall.test.ts packages/core/tests/integration/recall-tool.test.ts packages/core/tests/resources/project-map.test.ts`
- `npm run build`
- `npm run check`
- `npm audit --audit-level=moderate`
- Real Codex CLI runtime check with marker `TRACKA-LIVE-20260423`

Observed release-gate behavior:

- Codex CLI detected through `CODEX_HOME`.
- `memory_import_codex` imported live Codex events from rollout JSONL.
- `memory_search` found `TRACKA-LIVE-20260423` and the decision `SQLite cache + redacted capture`.
- `memory_recall` no longer returned `no_memory`; it returned inspectable candidates when multiple matching events existed.

## Known Limitations

- Codex desktop / extension behavior still depends on upstream MCP exposure and remains unverified for this release.
- Duplicate-heavy recall can still return `needs_clarification`; better ranking of direct user decisions above diagnostic/session-end echoes is future UX polish.
- Repo-local plugin packaging is available, but manual MCP setup remains the documented fallback until marketplace-based one-command install is proven.
