# Locus v3.6.1 Release Notes

`v3.6.1` is a focused Codex install hotfix for `v3.6.0`.

It carries forward the Track C richer recall work from `v3.6.0`: `memory_recall`, `candidateGroups`, redacted/full capture UX, durable extraction, `memory_review`, and audit visibility.

## Fixed

- The one-command installer now adds `CODEX_HOME` to the generated Locus MCP environment.
- Fresh Codex installs now start Locus with Codex client detection and use `$CODEX_HOME/memory/` for storage instead of the generic fallback `~/.locus/memory/` path.

## Validation Target

After publishing, validate with:

```powershell
npx -y locus-memory@3.6.1 install codex --yes
```

Then restart Codex and confirm `memory_status` reports Codex diagnostics and a database path under `.codex\memory`.

## Known Boundaries

- Codex CLI remains the validated primary path.
- Codex desktop / extension parity remains unverified until directly tested in that surface.
- `full` capture remains explicit warning territory; `redacted` is the recommended practical recall mode.
