# Locus v3.3.0

## Summary

`v3.3.0` is the first release where Codex support is treated as a shipped product line instead of an experimental side path. The release combines the Phase 1-7 Codex work into one validated workflow: import, auto-import before search, diagnostics, skill guidance, VS Code documentation, and repo-local plugin packaging.

## Highlights

- `memory_import_codex` is available as a stable manual catch-up path with `latestOnly`, `projectRoot`, `sessionId`, and `since` filters.
- Codex auto-import runs before `memory_search` in a bounded, debounced, best-effort path.
- `memory_status` and `memory_doctor` now expose Codex-specific diagnostics.
- The canonical Codex skill is synced through `npm run sync:codex-skill`.
- Repo-local plugin packaging is available through `plugins/locus-memory/`, `.agents/plugins/marketplace.json`, and `npm run sync:codex-plugin`.
- The Codex VS Code extension path is now documented with explicit upstream-boundary notes.

## Upgrade Notes

- Default Codex capture remains `metadata`.
- In `metadata` mode, `v3.3.0` validates event ingestion, diagnostics, and idempotency.
- Do not expect full semantic recall of Codex dialogue text in `metadata` mode.
- If you need richer conversational recall later, that belongs to a future release track built around `redacted` or `full`.

## Local Validation Summary

Validated locally on the release branch before GitHub push:

- `npm run build`
- `npm run sync:codex-skill`
- `npm run sync:codex-plugin`
- `memory_status` in a real Codex session
- `memory_doctor` in a real Codex session
- `memory_import_codex` via `since` against a contentful older Codex session
- repeated `memory_import_codex` idempotency check

Observed release-gate behavior:

- Codex diagnostics and manual import work in the real local Codex environment.
- `since` is currently the reliable acceptance selector for manual import verification.
- `sessionId` is not used as a release blocker for `v3.3.0`.
- `metadata` capture verifies ingestion and idempotency, not full text recall.

## Known Limitations

- Codex VS Code extension behavior still depends partly on upstream MCP exposure in preview builds.
- Repo-local plugin packaging improves onboarding, but manual MCP setup remains the documented fallback.
- Rich dialogue recall from Codex sessions is intentionally not the release gate for `v3.3.0` while default capture stays `metadata`.
