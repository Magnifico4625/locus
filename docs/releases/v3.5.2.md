# Locus v3.5.2 Release Notes

**Release focus:** Codex install diagnostics polish.

`v3.5.2` fixes a follow-up diagnostic issue after the `v3.5.1` startup hotfix.

## What Was Fixed

`locus-memory doctor codex` could report:

```text
Ownership: missing
```

even when the real Codex MCP config contained the package-owned Locus runtime:

```toml
[mcp_servers.locus]
command = "npx.cmd"
args = ["-y", "locus-memory@3.5.2", "mcp"]
cwd = "C:\\Users\\<you>\\.codex"
```

Root cause: `doctor codex` classified ownership only from an injected test reader. In the real CLI path no reader is injected, so it did not fall back to `codex mcp get locus`.

## Fix

`doctor codex` now uses the same fallback pattern as uninstall:

```text
codex mcp get locus
```

and classifies that output as `package-owned`, `manual-locus`, `foreign-locus`, or `missing`.

## Validation

- added a regression test for the real `codex mcp get locus` doctor path
- verified the targeted doctor test passes
- full release validation should include `npm run check`, `npm pack --dry-run`, registry metadata, and `npx` smoke checks before GitHub release publication
